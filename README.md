# 財務審核系統

這是一個獨立的新專案，與原本門市查詢系統分開。系統包含員工費用申請、LINE 群組審核連結、拒絕理由填寫、後台員工權限管理與財務審核資料管理。

## 功能

- 前台員工填寫費用申請表：員工、申請項目、金額、內容描述、備註等欄位。
- 前台費用類別由後台設定，不再寫死在程式內。
- 送出後寫入本機 JSON 資料庫，並可推送 LINE 群組訊息。
- LINE 訊息提供「同意」與「不同意」按鈕。
- 同意後，申請狀態改為已同意，後台財務清單會留下紀錄。
- 不同意時，主管需填寫理由，申請資料會保留理由供修正。
- 後台可新增、編輯、刪除員工，並設定權限為員工或管理員。
- 後台可新增、編輯、刪除單位；員工可同時隸屬多個單位，部門位於單位之下。
- 後台可新增、編輯、刪除費用類別。
- 後台可新增、編輯、刪除財務申請紀錄，並匯出 CSV。
- 前台與後台可拍照或上傳發票照片，主管審核與後台清單可查看附件。
- 後台支援管理員與員工登入。管理員可管理全部資料；員工只能查看自己的總覽與申請。
- 員工可編輯自己的未核准申請，若申請已同意核可，系統會鎖定不可修改。
- 員工可在後台「個人資料」分頁修改姓名、部門、登入帳號與密碼。

## 啟動

```bash
cd finance-approval-system
npm start
```

預設網址：

- 前台申請表：http://localhost:3100/
- 主管審核頁：由 LINE 按鈕連結開啟
- 後台管理：http://localhost:3100/admin

預設後台帳號：

- 帳號：`admin`
- 密碼：`admin12345`

員工登入帳號需由管理員在後台「員工權限」中替該員工設定帳號與密碼。

正式使用前請複製 `.env.example` 為 `.env`，並修改密碼與金鑰。

## GitHub + Render 部署

本系統需要 Node.js 後端、登入 session、LINE 推播、發票上傳與資料儲存，不適合只用 GitHub Pages。

建議正式部署方式：

1. GitHub 保存程式碼。
2. Render 建立 Node Web Service。
3. Render 環境變數保存密碼與 LINE token。
4. Render persistent disk 保存 `store.json` 與發票附件。

詳細步驟請看 [DEPLOYMENT.md](DEPLOYMENT.md)。

若單位仍有員工使用，系統會拒絕刪除，避免員工資料失去歸屬。

## LINE 設定

`.env` 需要設定：

```bash
APP_BASE_URL=https://你的公開網址
LINE_CHANNEL_ACCESS_TOKEN=你的 LINE Messaging API channel access token
LINE_TARGET_ID=LINE 群組 ID
```

注意事項：

- `APP_BASE_URL` 必須是主管手機可以開啟的公開網址，LINE 群組內的審核按鈕才會有效。
- `LINE_TARGET_ID` 可以透過 LINE Bot webhook 事件取得 groupId。取得後填入 `.env`。
- 如果沒有設定 LINE 參數，申請仍會儲存，系統會顯示 LINE 通知尚未啟用。

## 資料位置

資料會儲存在：

```text
data/store.json
```

發票照片會儲存在：

```text
data/invoices/
```

這些檔案包含員工清單、費用申請紀錄與發票附件。部署正式環境時，建議改接資料庫與正式檔案儲存，或把 `data` 放在持久化磁碟。
